About HomeShield

Project, detection stack and architecture credits.

HomeShield is a centralized dashboard for real-time CCTV monitoring and machine-learning-powered anomaly detection. One unified pipeline runs three GPU-accelerated detectors on every camera feed, draws bounding-box overlays on the live MJPEG stream, persists every alert to SQLite with an annotated snapshot, and pushes updates to this dashboard via Server-Sent Events.

Detection stack
  🤸 Fall — YOLO11 / YOLO26 pose estimation feeds a 7-state finite-state machine (Standing → Walking → Sitting → Fall_Detected → Lying_After_Fall → Lying_Motionless → Inactivity). Two-stage decision (peak descent velocity + sustained horizontal posture) rejects controlled sit-downs.
  🔥 Fire — custom YOLO weights covering fire and smoke classes, with per-class cooldown to debounce alerts.
  👤 Face — InsightFace ArcFace (buffalo_l) producing 512-dimensional embeddings, cosine-matched against the registered Persons gallery; unknown faces are auto-logged to the Intruders list.
  🚧 Zones — polygon zones per camera. Safe zones suppress lying / inactivity alerts (e.g. on a bed); danger zones trigger a child-entry alert.

Architecture
Per-camera worker threads share one set of YOLO / ONNX models loaded on GPU. Frame skipping (fire every 2nd frame, face every 5th) keeps pose running on every frame. Non-blocking event publishing offloads JPEG encoding + SQLite writes to a daemon thread so the capture loop never waits on disk I/O.

Tech stack
  • Backend: Python 3.11, PyTorch + CUDA, Ultralytics YOLO, InsightFace + ONNX Runtime GPU
  • Vision: OpenCV, NumPy
  • Web: Flask, MJPEG (multipart/x-mixed-replace), Server-Sent Events
  • Storage: SQLite (WAL mode), filesystem snapshots
